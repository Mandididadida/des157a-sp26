About Media Queries

Media queries are CSS rules that let a web page change its layout or styling depending on the size of the screen or browser window. They are useful because the same page may be viewed on a phone, tablet, or desktop, and each screen size needs a slightly different layout. A media query checks a condition, and if that condition is true, the styles inside it will be applied.

A min-width media query applies styles when the screen is at least a certain width, while a max-width media query applies styles when the screen is at or below a certain width. In many modern layouts, designers start with small screens first and then use min-width queries to add more layout for larger screens. This approach works better because it keeps the base design simpler and makes it easier to expand the layout step by step.

Relative Units

Relative units are measurements that change based on the screen, the parent element, or the text size. They are useful in web design because they make pages more flexible and responsive. For example, em units are often used for type because they scale based on the font size. Percentages are useful for widths because they let elements grow or shrink with the page layout.

Other relative units include vw and vh, which stand for viewport width and viewport height. These units are based on the size of the browser window, so they can be useful for banners, sections, or spacing that should respond to the screen size. Relative units help web pages adapt better than fixed pixel values in many situations.

Meta Viewport Tag

The meta viewport tag tells the browser how to scale and size the page on mobile devices. Without it, the browser may try to show the whole desktop layout on a small screen, which makes the text look tiny and the page hard to read. With the viewport tag included, the page matches the width of the device and the layout behaves more naturally on phones and tablets.

When I compare pages with and without the meta viewport tag in Chrome inspector, the version without it looks zoomed out and much smaller on a mobile screen. The version with it fills the screen better and is easier to read. This tag is important because it helps responsive CSS work the way it is supposed to.
